Задание №2

	Результат выводится в консоль.
	Увы, но при выводе результов промежуточной сортировки в консоль выводится конечный результат.
	Во время дебага кода результирующие объекты для каждой отдельной функции сортировки приводятся корректно,
	но стоит запустить выполнение без дебага, как тут же во всех промежуточных результатах будет один и тот же "конечный" объект.
	Как я понял, это связанно с тем, что сложные объекты копируются частично и их сложные внутренние части (объекты, массивы) ссылаются на "родителя".
	Промежуточные результаты можно получить путем поэтапного выполнения программы через дебаг.
	ВАЖНО! Для функции sortBy важна последовательность ключей (первая и главная сортировка будет проходить по первому ключу - sort_by[0]).

	Результаты прохождения выводятся в консоли.
	После этого выводятся все возможные варианты прохождения опроса (для удобства выбраны индексы задействованных вопросов).
	Затем выводится конечный объект, полями которого являются number и list. В первом свойстве хранится количество возможных комбинаций, во втором - сами комбинации
	с соответствующим для следующего вопроса ответом (если следующий вопрос не зависит от ответа, то передается оба ответа).
